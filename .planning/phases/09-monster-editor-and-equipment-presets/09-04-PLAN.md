**Three-Tier Color Highlighting on Preview Panel:**

Per locked decision, three distinct persistent colors for modification sources:

```python
STYLE_EQUIPMENT   = "color: #4EA8DE;"    # steel blue — equipment-modified values
STYLE_MANUAL      = "color: #F4A261;"    # amber — manually edited values
STYLE_CUSTOM_FLAG = "color: #E63946;"    # red — custom override (doesn't match prof math)
STYLE_BASE        = ""                   # no style — unmodified base values
```

After every `_rebuild_preview()` call, apply color highlighting to the preview panel:

1. Compare working_copy vs base_monster field by field
2. For each modified field, determine the source:
   - If the field was changed because of equipment (e.g. AC after armor, or an action's to-hit after weapon) → STYLE_EQUIPMENT
   - If the field was changed by MathValidator flagging it as CUSTOM (doesn't match non-prof/prof/expertise) → STYLE_CUSTOM_FLAG
   - If the field was manually edited by the user → STYLE_MANUAL
3. Apply the style to the corresponding QLabels in the preview panel via setStyleSheet()
4. Set tooltip on modified labels: `label.setToolTip(f"Base: {base_value}")` (per locked decision: tooltip only, keeps statblock clean)

**Implementation approach for highlighting:**
- Add a method `_apply_highlights(derived_stats)` called after `_rebuild_preview()`
- Track modification sources in a dict: `self._mod_sources: dict[str, str]` mapping field keys to source types ("equipment", "manual", "custom")
- When equipment changes a value → record source as "equipment"
- When user edits a spinbox/field → record source as "manual"
- When MathValidator flags a save/attack as CUSTOM → record source as "custom"
- The preview panel's show_monster() may need minor extensions to return QLabel references for highlightable fields. If MonsterDetailPanel doesn't expose individual labels, apply highlighting by finding child QLabels by objectName or by extending MonsterDetailPanel with a `get_highlight_labels() -> dict[str, QLabel]` method.
