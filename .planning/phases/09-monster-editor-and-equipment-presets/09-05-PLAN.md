**In src/ui/monster_editor.py:**

1. Update constructor to accept `library: MonsterLibrary` and `persistence: PersistenceService` parameters (passed from LibraryTab).

2. Implement `_save_override()`:
   - Build a MonsterModification from the diff between _base_monster and _working_copy:
     - Only populate fields that actually changed (don't store unchanged fields)
     - Include equipment list, buffs list, action overrides, skills, size, hp_formula as applicable
   - Replace the monster in MonsterLibrary: `self._library.replace(self._base_monster.name, self._working_copy)`
   - Save MonsterModification to persistence: `self._persistence.save_modified_monsters({...updated dict...})`
   - Emit `self.monster_saved.emit(self._working_copy)`
   - Set `self._dirty = False`
   - Accept the dialog: `self.accept()`

3. Implement `_save_as_copy()`:
   - Open QInputDialog.getText() with title "Save as Copy", label "Enter a name for the copy:", empty default text
   - Validate: name must not be empty, must not already exist in library (check `self._library.has_name(proposed_name)`). If collision, show QMessageBox error and re-prompt.
   - Set working_copy.name = proposed_name
   - Build MonsterModification with custom_name = proposed_name
   - Add to library: `self._library.add(self._working_copy)` (adds as new monster)
   - Save MonsterModification to persistence with custom_name as key
   - Emit `self.monster_saved.emit(self._working_copy)`
   - Set `self._dirty = False`
   - Accept the dialog

4. Wire the Save menu actions (created as stubs in Plan 03) to `_save_override` and `_save_as_copy`.

5. Update closeEvent Save button to call `_save_override()` (the default save action when closing).

**In src/ui/monster_table.py:**

1. Add `self._modified_names: set[str] = set()` attribute to MonsterTableModel.

2. Add `set_modified_names(names: set[str])` method that sets `_modified_names` and emits layoutChanged to refresh the table.

3. Update `data()` for column 3 (badge column):
   - If monster.name in `self._modified_names`: return a pencil icon character or unicode symbol (e.g. "✎" or a small icon)
   - Existing "!" for incomplete monsters takes priority (show "!" if incomplete, "✎" if modified, "" otherwise)

4. Optionally add a ToolTip role for column 3: "Incomplete statblock" for "!", "Modified" for "✎"

**In src/ui/library_tab.py:**

1. Update `_on_edit_monster(monster)` to pass `library` and `persistence` to MonsterEditorDialog constructor.

2. Connect `dialog.monster_saved` to a new method `_on_monster_saved(monster)`:
   - Refresh the table model: `self._model.reset_monsters(self._library.all())`
   - Update modified names: `self._model.set_modified_names(self._get_modified_names())`
   - Re-select the saved monster in the table if still present

3. Add `_get_modified_names() -> set[str]` that queries PersistenceService for all modified monster keys:
   - `return set(self._persistence.load_modified_monsters().keys())`

4. Accept `persistence: PersistenceService` as a constructor parameter (passed from MainWindow).

**In src/ui/app.py (MainWindow):**

1. Pass PersistenceService to MonsterLibraryTab constructor.

2. On startup after loading persisted modifications, populate modified names:
   - `self._library_tab._model.set_modified_names(set(self._persisted_modifications.keys()))`

3. Wire persisted modifications to actually apply on startup:
   - For each key in `_persisted_modifications`, reconstruct MonsterModification via `.from_dict()`, and if the base monster exists in library, apply the modification to create a modified Monster and replace it in the library.
   - This requires a helper: given a base Monster and a MonsterModification, produce a modified Monster with all overridden fields applied.

4. Wire auto-save to include modified_monsters:
   - `_save_persisted_data()` already saves all categories. Ensure `self._persisted_modifications` is kept in sync when MonsterEditorDialog saves (connect via signal or direct update).
