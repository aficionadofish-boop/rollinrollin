  <name>Task 1: Create MonsterEditorDialog with core editing sections</name>
  <files>src/ui/monster_editor.py</files>
  <action>
Create `src/ui/monster_editor.py` containing CollapsibleSection helper and MonsterEditorDialog.

**CollapsibleSection(QWidget):**
- Constructor takes `title: str`, `content: QWidget`, `expanded: bool = False`
- QPushButton header with "+/- {title}" toggle text
- Content QWidget hidden/shown on toggle click
- Follows the existing lore toggle pattern from monster_detail.py
- Add `set_summary(text: str)` method that appends a grey summary to the header button text when collapsed (for Equipment section use in Plan 04)

**MonsterEditorDialog(QDialog):**

**Constructor: `__init__(self, monster: Monster, parent=None)`**
- Store `self._base_monster = monster` (never mutated)
- Store `self._working_copy = copy.deepcopy(monster)` (edited in-place)
- Store `self._undo_stack: list = []` (snapshots for undo)
- Store `self._recalculating: bool = False` (signal guard)
- Store `self._dirty: bool = False` (unsaved changes flag)
- Set near-fullscreen size: 92% of parent.size() if parent, else 1000x700
- Set modal, window title "Editing: {monster.name}"

**Layout:**
- QVBoxLayout main with toolbar at top, then QSplitter below
- QSplitter horizontal: left scroll area (edit fields), right preview panel
- Left side: QScrollArea containing QVBoxLayout with collapsible sections
- Right side: Create a MonsterDetailPanel instance as the preview (reuse existing widget)

**Toolbar (QHBoxLayout):**
- Editable monster name QLineEdit (bold, large font) — updates working_copy.name and window title on editingFinished
- Stretch spacer
- Save QPushButton with QMenu dropdown: "Save (override base)", "Save as Copy..." — both are NO-OP stubs in this plan (wired in Plan 05)
- Discard QPushButton — calls `self._discard()`
- Undo QPushButton — calls `self._undo()`

**Section 1: Ability Scores (expanded by default)**
- 6-across grid: STR, DEX, CON, INT, WIS, CHA
- Each: QLabel header, QSpinBox (range 1-30, value from working_copy.ability_scores)
- Connect each spinbox editingFinished to `_on_ability_changed`
- `_on_ability_changed`: guard with `_recalculating`, push undo, update working_copy, rebuild preview

**Section 2: Saving Throws (collapsed by default)**
- One row per save (STR, DEX, CON, INT, WIS, CHA)
- Each row: QLabel(ability name), toggle buttons (Non-Prof / Prof / Expertise / Custom), QSpinBox for custom override (enabled only when Custom selected)
- Initialize from working_copy.saves + MathValidator.validate_saves() to detect current state
- On toggle change: push undo, update working_copy.saves with computed value (mod, mod+prof, mod+2*prof, or custom spinbox value), rebuild preview

**Section 3: Skills (collapsed by default)**
- Show only skills the monster currently has (non-empty skills dict)
- Each row: QLabel(skill name), toggle (Non-Prof / Prof / Expertise / Custom), QSpinBox for custom override
- "Add Skill" button at bottom — QComboBox with all 18 skills from SKILL_TO_ABILITY, adds a new row on selection
- Remove button (X) per row to delete a skill
- On change: push undo, update working_copy.skills, rebuild preview

**Section 4: Hit Points (collapsed by default)**
- QLineEdit for hit dice formula (e.g. "7d8+14") — validation: accept any non-empty string
- QSpinBox for flat max HP (range 1-9999)
- Both editable independently — DM can set formula and/or flat HP
- On change: push undo, update working_copy.hp, rebuild preview

**Section 5: Challenge Rating (collapsed by default)**
- QComboBox with all CR values: "0", "1/8", "1/4", "1/2", "1" through "30"
- On change: push undo, update working_copy.cr, recalculate (cascades prof bonus to all derived values), rebuild preview

**_rebuild_preview() method:**
- Call MonsterMathEngine().recalculate(self._working_copy) to get DerivedStats
- Call self._preview_panel.show_monster(self._working_copy) to refresh the statblock display
- Update window title with current monster name

**_push_undo() method:**
- Append copy.deepcopy(self._working_copy) to self._undo_stack
- Set self._dirty = True

**_undo() method:**
- If undo_stack not empty: pop last snapshot, assign to self._working_copy, repopulate all form fields from working_copy, rebuild preview

**_discard() method:**
- If dirty, show QMessageBox "Discard all changes?" with Yes/No
- If Yes: reject() the dialog (closes it, returns Rejected)

**closeEvent(event) override:**
- If dirty: show QMessageBox with Save/Discard/Cancel buttons
- Save → stub (Plan 05 wires this)
- Discard → event.accept()
- Cancel → event.ignore()
- If not dirty: event.accept()

**reject() override:**
- Route through close() to ensure closeEvent guard fires

**_populate_form() method:**
- Populate all form widgets from self._working_copy values
- Use blockSignals(True/False) around all programmatic widget updates to prevent signal re-entry

**Signals:**
- `monster_saved = Signal(object)` — emitted when save completes (Plan 05 connects)
