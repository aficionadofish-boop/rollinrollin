**MonsterMathEngine** (src/monster_math/engine.py):

`_PROF_BY_CR` lookup table — string keys matching Monster.cr format:
- "0","1/8","1/4","1/2","1"-"4" → 2
- "5"-"8" → 3; "9"-"12" → 4; "13"-"16" → 5; "17"-"20" → 6; "21"-"24" → 7; "25"-"28" → 8; "29"-"30" → 9

`_ability_mod(score: int) -> int` — `(score - 10) // 2` (floor division, same formula as encounter/service.py)

`DerivedStats` dataclass:
- proficiency_bonus: int
- ability_modifiers: dict[str, int] — {"STR": 3, "DEX": 1, ...}
- expected_saves: dict[str, int] — mod only (non-proficient baseline)
- expected_proficient_saves: dict[str, int] — mod + prof
- expected_expertise_saves: dict[str, int] — mod + 2*prof

`MonsterMathEngine.recalculate(monster: Monster) -> DerivedStats` — Pure function. Reads monster.cr for proficiency, monster.ability_scores for mods. Returns complete DerivedStats. Never mutates input.

Test cases:
- Monster with STR=16, CR="5" → mod=3, prof=3, expected_save_STR=3, prof_save=6, expertise_save=9
- Monster with DEX=8 → mod=-1
- Monster with CR="1/2" → prof=2
- Monster with CR="30" → prof=9
- Monster with no ability_scores → all mods default to 0

**MathValidator** (src/monster_math/validator.py):

`SaveState(str, Enum)` — NON_PROFICIENT, PROFICIENT, EXPERTISE, CUSTOM (use str,Enum for JSON serialization)

`SaveValidation` dataclass: ability, actual, expected_non_proficient, expected_proficient, expected_expertise, state: SaveState, is_flagged property (True when CUSTOM), tooltip property (shows expected vs actual breakdown)

`ActionValidation` dataclass: action_name, actual_to_hit, expected_to_hit (ability_mod + prof), to_hit_delta, to_hit_is_flagged property, actual_damage_bonus (int|None), expected_damage_bonus (ability_mod only — damage doesn't add proficiency), damage_delta, damage_is_flagged property. Overall is_flagged = to_hit_is_flagged or damage_is_flagged

`SpellValidation` dataclass: trait_name, actual_attack_bonus, expected_attack_bonus, actual_dc (optional), expected_dc (optional), delta_attack, delta_dc (optional), is_flagged property

`MathValidator.validate_saves(monster, derived) -> list[SaveValidation]` — For each ability in monster.saves, compare actual vs three expected values, classify as NON_PROFICIENT/PROFICIENT/EXPERTISE/CUSTOM.

`MathValidator.validate_action(action, monster, derived) -> ActionValidation` — Compare action.to_hit_bonus against (ability_mod + prof) AND action.damage_bonus against (ability_mod alone — damage bonus does NOT include proficiency). The relevant ability for both checks uses the same heuristic: if the action name/raw_text contains "Ranged" use DEX; else if "finesse" in raw_text and DEX mod > STR mod use DEX; else use STR. If action.to_hit_bonus is None, skip to-hit validation (non-attack action). If action.damage_bonus is None, skip damage validation (action has no damage bonus field). Both are validated independently — an action can have correct to-hit but flagged damage, or vice versa.

`MathValidator.validate_spellcasting(spellcasting_info, monster, derived, actual_attack_bonus=None, actual_dc=None) -> SpellValidation` — Expected attack = casting_mod + prof + focus_bonus. Expected DC = 8 + casting_mod + prof + focus_bonus.

Test cases:
- Save STR +3 on monster with STR=16, CR=5 → PROFICIENT (mod 3 + prof 3 = 6... wait, +3 is just mod, so NON_PROFICIENT)
- Save STR +6 on same monster → PROFICIENT (3 + 3)
- Save STR +9 on same monster → EXPERTISE (3 + 2*3)
- Save STR +7 on same monster → CUSTOM (tooltip: "Expected +6 (prof + STR), got +7 — custom +1")
- Action to_hit=+6, damage_bonus=+3 on STR=16 CR=5 monster → to-hit expected +6 (mod 3 + prof 3) not flagged, damage expected +3 (mod only) not flagged
- Action to_hit=+8, damage_bonus=+3 on same → to-hit flagged (delta +2), damage not flagged
- Action to_hit=+6, damage_bonus=+5 on same → to-hit not flagged, damage flagged (expected +3, got +5, delta +2)
- Action to_hit=+8, damage_bonus=+5 on same → both flagged
- Spell attack +7 with INT casting (INT=18, mod=4), CR=5 (prof=3), focus=0 → expected +7, not flagged
- Spell DC 15 with INT=18, CR=5, focus=0 → expected 8+4+3=15, not flagged

**SpellcastingDetector** (src/monster_math/spellcasting.py):

`SpellcastingInfo` dataclass: trait_name, casting_ability, is_assumed, focus_bonus=0

NOTE: This SpellcastingInfo is defined in the spellcasting module as a runtime detection result, separate from the domain model version. Import from here for math operations; the domain model version in models.py is for persistence serialization.

`SpellcastingDetector.detect(monster: Monster) -> list[SpellcastingInfo]` — Scan monster.actions for entries whose name contains "spellcasting" (case-insensitive). For each match, extract casting ability from raw_text using regex `\b(Strength|Dexterity|Constitution|Intelligence|Wisdom|Charisma)\b`. Map to abbreviation (INT, WIS, CHA, etc.). If not found, fallback to highest mental stat (WIS, INT, CHA) from monster.ability_scores, set is_assumed=True.

Test cases:
- Monster with action named "Spellcasting" and raw_text "Intelligence (spell save DC 15)" → SpellcastingInfo(trait_name="Spellcasting", casting_ability="INT", is_assumed=False)
- Monster with "Innate Spellcasting" and raw_text "Charisma" → casting_ability="CHA"
- Monster with "Spellcasting" but no ability in text, WIS=16, INT=14, CHA=12 → fallback to WIS (highest mental), is_assumed=True
- Monster with "Spellcasting" and "Innate Spellcasting" → returns 2 SpellcastingInfo entries
- Monster with no spellcasting actions → returns empty list
- Monster with action named "Multiattack" → not detected as spellcasting
