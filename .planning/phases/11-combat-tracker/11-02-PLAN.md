**src/ui/hp_bar.py** — Custom HP bar widget:

Create `HpBar(QWidget)` with:
- Constructor takes `max_hp: int`, `current_hp: int`, `temp_hp: int = 0`
- `clicked = Signal()` — emitted on mouse click (to reveal damage input)
- `setFixedHeight(24)` — consistent bar height
- `paintEvent(self, event)`:
  - Draw dark gray background (`#333`) filling full widget width
  - Calculate `hp_pct = current_hp / max_hp` (guard division by zero)
  - Color thresholds: `> 0.5` = green (`#4CAF50`), `> 0.25` = yellow (`#FFC107`), else red (`#F44336`)
  - Draw HP segment: `fillRect(0, 0, hp_width, h, color)` where `hp_width = int(w * hp_pct)`
  - Draw temp HP as blue (`#2196F3`) segment immediately to the right of the HP segment: `fillRect(hp_width, 0, temp_width, h, blue)` where `temp_pct = min(temp_hp / max_hp, 1.0)`. Only draw if temp_hp > 0.
  - Draw text overlay centered: `"{current_hp} / {max_hp}"` if no temp HP, or `"{current_hp} + {temp_hp}t / {max_hp}"` if temp HP > 0. White text, bold, with a slight shadow for readability.
  - Use `QPainter.setRenderHint(QPainter.RenderHint.Antialiasing)`
- `update_hp(current_hp: int, max_hp: int, temp_hp: int = 0)` — update stored values and call `self.update()` to trigger repaint
- `mousePressEvent(event)` — emit `clicked` signal
- Cursor: set `setCursor(Qt.CursorShape.PointingHandCursor)` to indicate clickability

  - `get_entries() -> list[str]` — return all log entries as a list of strings for persistence

**src/ui/combat_tracker_tab.py** — Main tab widget:

Create `CombatTrackerTab(QWidget)`:
- Signals:
  - `send_to_saves = Signal(list)` — list of selected combatant data for Save Roller (COMBAT-14, wired in Plan 04)

- Constructor `__init__(self, roller, library, parent=None)`:
  - Store `self._roller = roller`, `self._library = library`
  - Create `self._service = CombatTrackerService()`
  - Create `self._cards: dict[str, CombatantCard] = {}` — maps combatant_id to card widget
  - `self._selected_ids: set[str] = set()` — tracks multi-select (Plan 04 implements full multi-select)
  - Build layout via `_build_layout()`
  - `self._combat_active = False` — tracks whether combat has been started

- `_build_layout()`:
  - **Main layout**: QVBoxLayout
  - **Toolbar** (QHBoxLayout):
    - "Start Combat" button — loads encounter from sidebar data and initializes combat. When combat is active, button text changes to "Reload Encounter" (per research open question recommendation: explicit start, not auto-populate)
    - "Roll Initiative" button — calls `_on_roll_initiative()` (disabled until combat started)
    - "Reset Combat" button — calls `_on_reset_combat()` with QMessageBox.question confirmation (disabled until combat started)
    - Round counter: QLabel "Round 1" (bold, larger font)
    - "Next Turn" / "Previous Turn" buttons — initially hidden; shown by Plan 03 when initiative mode is wired
    - "Pass 1 Round" button — initially hidden; shown by Plan 03 when initiative mode is off
    - Stretch
    - "AOE Damage" button — disabled until multi-select is implemented in Plan 04
    - "Send to Saves" button — disabled until Plan 04 wiring
    - Gear/filter icon button for toggleable stats menu — placeholder; Plan 03 implements
  - **Body** (QSplitter, horizontal):
    - Left (70%): QScrollArea containing a QVBoxLayout (`self._card_layout`) for combatant cards. Set `widgetResizable = True`.
    - Right (30%): CombatLogPanel (`self._log_panel`)

- `start_combat(members: list[tuple])` — public method called by MainWindow:
  - `self._service.load_encounter(members, self._roller)`
  - Call `_rebuild_cards()` to create CombatantCard widgets
  - Set `self._combat_active = True`, enable toolbar buttons
  - Log "Combat started with {N} combatants"

- `_rebuild_cards()` — clear `self._card_layout`, destroy old card widgets. For each combatant in `self._service.state.combatants`, create a `CombatantCard(state)`, wire signals:
  - `card.damage_entered` → `_on_damage(combatant_id, value)`
  - `card.condition_add_requested` → `_on_condition_add_requested(combatant_id)`
  - `card.initiative_changed` → `_on_initiative_changed(combatant_id, value)`
  - `card.condition_clicked` → `_on_condition_clicked(combatant_id, condition_name)`
  - Add card to layout and to `self._cards` dict.

- `_refresh_cards()` — for each combatant in service state, call `self._cards[id].refresh(state)`. Update round counter label. This does NOT rebuild — just updates display values.

- `_on_damage(combatant_id: str, value: int)` — call `self._service.apply_damage(combatant_id, value)`, add log entry, refresh cards.

- `_on_condition_add_requested(combatant_id: str)` — show a QMenu or small popup with:
  - Section header "Conditions" → list of STANDARD_CONDITIONS with names
  - Section header "Buffs/Effects" → list of COMMON_BUFFS with names
  - Separator
  - "Custom..." entry → opens a small QDialog with name field + optional duration QSpinBox (0 = indefinite)
  - When a preset is selected, create ConditionEntry with preset default_duration. When custom, use user input.
  - Call `self._service.add_condition(combatant_id, condition_entry)`, log, refresh.
  - If multiple combatants are selected (Plan 04), apply to all selected.

- `_on_condition_clicked(combatant_id: str, condition_name: str)` — show small popup (QMenu):
  - "Edit Duration" → small inline QSpinBox dialog
  - "Remove" → call `self._service.remove_condition(combatant_id, condition_name)`, log, refresh
  - "Dismiss" (for expired) → same as remove

- `_on_initiative_changed(combatant_id: str, value: int)` — call `self._service.set_initiative(combatant_id, value)`. If initiative mode, rebuild card order to reflect new sort.

- `_on_roll_initiative()` — call `self._service.roll_all_initiative(self._roller)`, log entries, rebuild cards (since order changed).

- `_on_reset_combat()` — `QMessageBox.question(self, "Reset Combat", "Reset all HP, conditions, and initiative?")`. If yes, `self._service.reset_combat()`, log "Combat reset", rebuild cards.

- `load_combat_state(state_dict: dict)` — restore from persistence: `self._service.load_state(CombatState.from_dict(state_dict))`, rebuild cards, restore log entries.

- `get_combat_state() -> dict` — `self._service.state.to_dict()` with log entries from log panel.
