Add `GroupCard(QFrame)` class to `src/ui/combatant_card.py` (same file as CombatantCard):

**GroupCard** represents a collapsed group of same-type monsters (e.g., "4x Goblin"). Three-level progressive disclosure per locked decision:
1. **Collapsed** (default): Single card showing "Nx [Monster]" with average HP bar showing overall group health
2. **Expanded compact sub-rows**: Indented mini-rows for each individual (name, HP bar, conditions) — no AC/initiative
3. **Individual full card**: Click a sub-row to expand it into a full CombatantCard

Constructor: `__init__(self, group_id: str, members: list[CombatantState], parent=None)`:
- `self._group_id = group_id`
- `self._members = members`
- `self._expanded = False` — tracks collapsed/expanded state
- `self._expanded_individual: str | None = None` — ID of individually expanded member
- Build collapsed view by default

**Collapsed view layout** (QHBoxLayout):
- Count label: QLabel "Nx" where N is len(members), bold
- Monster name: QLabel (the shared monster name)
- AC badge: QLabel (all members share AC)
- Initiative: QSpinBox (shared initiative value)
- Average HP bar: HpBar showing avg(current_hp)/avg(max_hp) across all members. This is a VIEW computed from individuals.
- Expand button: QPushButton with down arrow icon/text. Click → `_toggle_expand()`
- Average condition summary: If any member has conditions, show a small indicator

**Expanded view layout** (QVBoxLayout):
- Header row identical to collapsed but with up arrow on expand button
- For each member in `self._members`:
  - **CompactSubRow** — a smaller QFrame (indented 20px left margin):
    - QHBoxLayout: individual name ("Goblin 1"), compact HpBar (smaller height 16px), condition chips (small), expand-to-full button
    - Click handler: if `self._expanded_individual == member.id`, collapse back to sub-row; otherwise expand to full CombatantCard

When an individual sub-row is clicked to expand:
- Replace the sub-row with a full `CombatantCard` (indented to show group membership)
- The full card emits the same signals as a standalone CombatantCard

**Signals** (pass through from individual CombatantCards):
- `damage_entered = Signal(str, int)` — from any individual
- `condition_add_requested = Signal(str)` — from any individual
- `condition_clicked = Signal(str, str)` — from any individual
- `initiative_changed = Signal(str, int)` — from group (applies to all members)

**Methods:**
- `refresh(members: list[CombatantState])` — update stored members and rebuild display
- `set_active_turn(active: bool)` — apply active styling to the group header (if any member is the active turn, the group header shows active)
- `set_member_active(combatant_id: str)` — highlight a specific member's sub-row within the expanded group
- `_toggle_expand()` — switch between collapsed and expanded views
- `_compute_avg_hp() -> tuple[int, int, int]` — return (avg_current, avg_max, avg_temp) from members

Also add `CompactSubRow(QFrame)` class — small inline row for expanded group view:
- Constructor: `__init__(self, state: CombatantState, parent=None)`
- Compact layout: name label, small HpBar (height 16px), condition chip summary, expand button
- `clicked = Signal(str)` — emits combatant_id when clicked for full expansion
- `refresh(state: CombatantState)` — update display

**Drag-to-reorder support on CombatantCard:**
- Add `setProperty("combatant_id", self._combatant_id)` for identification during drag
- The actual drag handling is on the scroll area in CombatTrackerTab (see Task 2)

**Update src/ui/combat_tracker_tab.py:**

**1. Initiative Mode Toggle**

Add a `QToolButton` or `QPushButton` "Initiative Mode" toggle in the toolbar:
- Checked = initiative mode ON: combatants sorted by initiative desc; show "Next Turn" and "Previous Turn" buttons; hide "Pass 1 Round" button
- Unchecked = initiative mode OFF: combatants in insertion order (or drag-to-reorder); hide Next/Previous Turn buttons; show "Pass 1 Round" button
- Connect to `_on_initiative_mode_toggled(checked: bool)` — update `self._service.state.initiative_mode`, show/hide relevant buttons, re-sort if toggling on

Wire the Next Turn, Previous Turn, and Pass 1 Round buttons that were created as placeholders in Plan 02:
- "Next Turn" → `_on_next_turn()`
- "Previous Turn" → `_on_previous_turn()`
- "Pass 1 Round" → `_on_pass_one_round()`

**2. Turn Cycling**

`_on_next_turn()`:
1. Call `self._service.advance_turn()` — returns list of log entries
2. Add log entries to combat log panel (set round number first)
3. Update round counter label
4. Call `_refresh_cards()` to update all card display
5. Update active turn highlighting: call `set_active_turn(False)` on all cards, then `set_active_turn(True)` on the card at the new current_turn_index
6. Auto-scroll to the active combatant card: `self._scroll_area.ensureWidgetVisible(active_card)`
7. Handle regeneration auto-apply if enabled (check service state)

`_on_previous_turn()`:
1. Call `self._service.undo_advance()` — returns bool
2. If True: add log entry "Turn undone", refresh cards, update highlighting
3. If False: show status message "Nothing to undo"

`_on_pass_one_round()`:
1. Call `self._service.pass_one_round()` — returns log entries
2. Add entries to log panel, update round counter, refresh cards

**3. Active Turn Visual Indicator**

After any turn change, iterate all cards/groups:
- Clear all `set_active_turn(False)`
- Find the card matching `self._service.state.combatants[self._service.state.current_turn_index].id`
- Call `set_active_turn(True)` on that card
- If the active combatant is inside a GroupCard, call `group.set_member_active(combatant_id)`

**4. Grouped Initiative Display**

Update `_rebuild_cards()` to support grouping:
- If `self._service.state.grouping_enabled`:
  - Group combatants by `group_id`
  - For each group with >1 member: create a `GroupCard(group_id, members)`
  - For each group with 1 member: create a regular `CombatantCard(state)`
  - Wire all signals from GroupCards the same way as CombatantCards
- If grouping disabled: create individual CombatantCards for all combatants

Add "Group" toggle button in toolbar:
- Checked = auto-grouping ON (default per locked decision)
- Unchecked = individual display
- Connect to `_on_grouping_toggled(checked: bool)` — updates `self._service.state.grouping_enabled`, calls `_rebuild_cards()`

**5. Toggleable Stat Display Menu**

Add gear/filter icon button in toolbar (was placeholder from Plan 02):
- Click opens a QMenu with checkable QActions:
  - "Walking Speed" (checked by default = hidden by default; user checks to show)
  - "Flying Speed"
  - "Climbing Speed"
  - "Swimming Speed"
  - "Burrowing Speed"
  - "Passive Perception"
  - "Legendary Resistance"
  - "Legendary Actions"
  - "Regeneration"
- Each action connects to `_on_stat_toggle_changed(stat_key: str, visible: bool)`
- Store toggle state in `self._visible_stats: dict[str, bool]`
- When toggled, iterate all cards and call `card.set_stat_visible(stat_key, visible)`

Add `set_stat_visible(stat_key: str, visible: bool)` to CombatantCard — shows/hides the corresponding label in the top row. For legendary resistance, show "LR: {remaining}/{max}". For legendary actions, show "LA: {remaining}/{max}". For regeneration, show "Regen: {hp}".

Also add regeneration toggles:
- "Regen Display" — show/hide regeneration info on cards
- "Regen Auto-Apply" — automatically apply regeneration healing when turn advances (stored in service)

**6. Drag-to-Reorder (Manual Mode)**

When initiative mode is OFF (manual reorder mode):
- Enable drag-and-drop on the scroll area's content widget
- On CombatantCard: implement `mousePressEvent/mouseMoveEvent` to start a QDrag with MIME type `application/x-combatant-id` containing the combatant_id
- On the card layout container: implement `dragEnterEvent/dropEvent` to accept the drop and reorder
- After drop, update the combatant order in `self._service.state.combatants` to match the new visual order
- When initiative mode is ON, disable drag-and-drop (sort is automatic)

**Update src/combat/service.py:**

Add `reorder_combatants(ordered_ids: list[str])` method:
- Reorder `self._state.combatants` to match the given ID order
- Update `current_turn_index` to track the same combatant
- Only works when `initiative_mode` is False

Add `set_auto_regen(enabled: bool)` method and `self._auto_regen: bool = False` field.

In `advance_turn()`, after condition decrement and before advancing the index, check if `_auto_regen` and combatant has `regeneration_hp > 0`: if so, call `apply_damage(combatant_id, +regeneration_hp)` and add log entry.
