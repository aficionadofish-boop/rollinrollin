**src/combat/models.py** — Create dataclasses:

1. `ConditionEntry` dataclass:
   - `name: str` — condition name (e.g. "Poisoned", "Bless")
   - `duration: int | None` — None = indefinite; positive int = rounds remaining
   - `expired: bool = False` — True when duration reaches 0
   - `color: str = ""` — hex color for UI chip (set by UI layer, not service)

2. `CombatantState` dataclass:
   - `id: str` — unique within combat, e.g. "Goblin_1"
   - `name: str` — display name, e.g. "Goblin 1"
   - `monster_name: str | None` — base monster name for library resolution; None for PCs
   - `max_hp: int`
   - `current_hp: int`
   - `temp_hp: int = 0`
   - `initiative: int = 0`
   - `ac: int = 0`
   - `conditions: list[ConditionEntry] = field(default_factory=list)`
   - `is_pc: bool = False`
   - `group_id: str = ""` — shared by same monster type for grouping, e.g. "Goblin"
   - `dex_score: int = 10` — for initiative tiebreaking (DEX score, not modifier)
   - `speed: str = ""` — e.g. "30 ft." (for toggleable display)
   - `passive_perception: int = 0` — for toggleable display
   - `legendary_resistances: int = 0` — uses remaining (auto-detected from traits)
   - `legendary_resistances_max: int = 0` — total uses (e.g. 3)
   - `legendary_actions: int = 0` — uses remaining
   - `legendary_actions_max: int = 0` — total uses
   - `regeneration_hp: int = 0` — HP per round (0 = no regen)
   - Property `is_defeated -> bool`: returns `current_hp <= 0`
   - `to_dict() -> dict` and `@classmethod from_dict(cls, d: dict) -> CombatantState` using known-field filtering pattern (match MonsterModification.from_dict style)

3. `CombatState` dataclass:
   - `combatants: list[CombatantState]`
   - `round_number: int = 1`
   - `current_turn_index: int = 0`
   - `initiative_mode: bool = True` — True = sorted by initiative; False = pass-1-round mode
   - `grouping_enabled: bool = True`
   - `log_entries: list[str] = field(default_factory=list)`
   - `to_dict() -> dict` and `@classmethod from_dict(cls, d: dict) -> CombatState`

4. `PlayerCharacter` dataclass:
   - `name: str`
   - `ac: int = 10`
   - `max_hp: int = 1`
   - `current_hp: int = 1`
   - `conditions: list[ConditionEntry] = field(default_factory=list)`
   - `to_dict() -> dict` and `@classmethod from_dict(cls, d: dict) -> PlayerCharacter`

5. `STANDARD_CONDITIONS: list[dict]` — constant list of 14 standard 5e conditions with name and default_duration:
   Blinded(None), Charmed(None), Deafened(None), Frightened(None), Grappled(None), Incapacitated(None), Invisible(None), Paralyzed(None), Petrified(None), Poisoned(None), Prone(None), Restrained(None), Stunned(None), Unconscious(None)

6. `COMMON_BUFFS: list[dict]` — constant list with name and default_duration:
   Bless(10), Shield(1), Bane(10), Hypnotic Pattern(10), Fear(10), Web(10), Entangle(10), Maze(None), Haste(10), Slow(10), Faerie Fire(10), Hold Person(10), Hex(None), Hunter's Mark(None), Spirit Guardians(10)

**src/combat/service.py** — CombatTrackerService class:

Constructor: `__init__(self)` — creates empty `CombatState()` and `_prev_snapshot: dict | None = None` (one-deep undo).

Methods:
- `load_encounter(members: list[tuple], roller)` — takes [(Monster, count)] from sidebar. For each monster, expand count into individual CombatantState entries with auto-numbered names ("Goblin 1", "Goblin 2"), populating max_hp, current_hp, ac, dex_score, group_id, speed, passive_perception from Monster fields. Also detect legendary resistance/actions/regeneration from Monster.actions raw_text using keyword scan: search for "Legendary Resistance" to extract uses count, "Legendary Actions" for uses, "Regenerates" or "Regeneration" for HP amount. Set `initiative = 0` (not yet rolled). Clear log. Set round_number=1, current_turn_index=0.
- `add_pcs(pcs: list[PlayerCharacter])` — add PC combatants with is_pc=True. Auto-number only if duplicate names.
- `remove_combatant(combatant_id: str)` — remove by ID. If removed index <= current_turn_index, decrement current_turn_index (clamp to 0). Per pitfall #2 from research.
- `roll_all_initiative(roller)` — for each non-PC combatant: `roller.roll_dice(1, 20).total + (dex_score - 10) // 2`. For grouped combatants (same group_id, grouping_enabled), all members get the same initiative (roll once per group). Sort combatants by initiative descending, then dex_score descending as tiebreaker. Log "Initiative rolled" entry.
- `apply_damage(combatant_id: str, signed_value: int) -> str` — if positive: heal (cap at max_hp). If negative: temp_hp absorbs first, remainder from current_hp (floor at 0). Return log entry string describing the change. Per COMBAT-15 and research code example.
- `apply_aoe_damage(combatant_ids: list[str], damage: int) -> list[str]` — call apply_damage for each ID with the same negative value. Return list of log entries.
- `add_condition(combatant_id: str, condition: ConditionEntry) -> str` — add to combatant's conditions list. Return log entry.
- `remove_condition(combatant_id: str, condition_name: str) -> str` — remove first matching condition by name. Return log entry.
- `advance_turn() -> list[str]` — (1) snapshot current state for undo (turn_index, round_number, and all condition durations). (2) Decrement timed conditions on combatant whose turn is ENDING (current_turn_index). Mark expired=True when duration hits 0. (3) If regeneration_hp > 0 and auto-regen is on, apply healing. (4) Advance current_turn_index (wrap). (5) If wrapped to 0, increment round_number. (6) Reset legendary_actions to max for the combatant whose turn is NOW starting. (7) Return log entries for all changes.
- `undo_advance() -> bool` — restore from snapshot (turn_index, round_number, all condition durations). Return False if no snapshot. Per research pattern 7.
- `pass_one_round() -> list[str]` — decrement all timed conditions on ALL combatants by 1. Increment round_number. Return log entries.
- `set_initiative(combatant_id: str, value: int)` — manual initiative edit. Re-sort if initiative_mode is on.
- `reset_combat() -> None` — reset all HP to max, clear conditions, clear initiative, round_number=1, clear log.
- `get_combatant(combatant_id: str) -> CombatantState | None`
- `@property state -> CombatState` — read-only access to current state
- `load_state(state: CombatState)` — restore from persistence
- `_find(combatant_id: str) -> CombatantState` — internal lookup by id, raise ValueError if not found
- `_sort_by_initiative()` — sort combatants by (initiative DESC, dex_score DESC). Preserve current_turn_index by tracking the active combatant's ID across the sort.
