**Multi-Select — in CombatTrackerTab:**

Implement three selection modes per locked decision:

1. **Ctrl-click**: On CombatantCard, detect `event.modifiers() & Qt.KeyboardModifier.ControlModifier` in the card's `mousePressEvent`. Toggle the card's combatant_id in `self._selected_ids`. Emit a `card_clicked = Signal(str, object)` (combatant_id, QMouseEvent) that CombatTrackerTab handles to update selection state.

2. **Shift-click**: Track `self._last_selected_id: str | None`. On Shift+click, select all combatants between `_last_selected_id` and clicked card in the current display order.

3. **Box drag selection** using QRubberBand:
   - On the combatant scroll area viewport, override `mousePressEvent/mouseMoveEvent/mouseReleaseEvent`:
     - `mousePressEvent`: if left button and no modifier, record `_drag_origin = event.pos()`, create `QRubberBand(QRubberBand.Shape.Rectangle, self.viewport())`, show it
     - `mouseMoveEvent`: update rubber band geometry from drag_origin to current pos (normalized)
     - `mouseReleaseEvent`: hide rubber band, compute intersection with all card widget geometries, add all intersecting card IDs to `self._selected_ids`
   - Per research pattern 6: use `QRubberBand` standard widget

   Create a `CombatantListArea(QScrollArea)` subclass that handles the rubber band logic. Replace the plain QScrollArea in CombatTrackerTab with this subclass.

4. **Selection visual**: Selected cards get blue border (`border: 2px solid #2196F3`). Active turn has golden glow. Active turn takes priority over selection visual if both apply.

5. **Bulk actions on selection**:
   - When `_on_condition_add_requested(combatant_id)` is called and `len(self._selected_ids) > 1`: apply the added condition to ALL selected combatants
   - When `_on_damage(combatant_id, value)` is called and `len(self._selected_ids) > 1`: apply damage to ALL selected combatants (this is manual multi-damage, distinct from AOE)
