4. Add a method to generate the auto-name:
   ```python
   def _generate_auto_name(self) -> str:
       import datetime
       ts = datetime.datetime.now().strftime("%Y-%m-%d %H:%M")
       count = sum(c for _, c in self.get_members()) if self.get_members() else 0
       return f"{ts} \u2014 {count} creature{'s' if count != 1 else ''}"
   ```

5. Store the current auto-name separately: `self._current_auto_name = ""`. When the encounter changes (creatures added/removed), update the auto-name. But **only** update `_name_edit.setText()` if the field currently contains the previous auto-name (not a custom name):
   ```python
   def _update_auto_name(self):
       new_auto = self._generate_auto_name()
       if self._name_edit.text() == self._current_auto_name or self._name_edit.text() == "":
           self._name_edit.setText(new_auto)
       self._current_auto_name = new_auto
   ```
   Call `_update_auto_name()` whenever the encounter membership changes (connect to the relevant signal — likely the same place that updates `_summary_label`).

   def get_save_name(self) -> str:
