**SettingsService (`src/settings/service.py`):**

```python
from __future__ import annotations
import dataclasses
import json
from pathlib import Path
from src.settings.models import AppSettings

_FILENAME = "settings.json"

class SettingsService:
    def __init__(self, workspace_root: Path) -> None:
        self._path = workspace_root / _FILENAME

    def load(self) -> AppSettings:
        if not self._path.exists():
            return AppSettings()
        try:
            data = json.loads(self._path.read_text(encoding="utf-8"))
        except (json.JSONDecodeError, OSError):
            return AppSettings()
        known = {f.name for f in dataclasses.fields(AppSettings)}
        filtered = {k: v for k, v in data.items() if k in known}
        return AppSettings(**filtered)

    def save(self, settings: AppSettings) -> None:
        data = dataclasses.asdict(settings)
        self._path.write_text(json.dumps(data, indent=2), encoding="utf-8")
```

Key implementation notes:
- Use `dataclasses.fields(AppSettings)` (public API), NOT `AppSettings.__dataclass_fields__` per Pitfall 4 in RESEARCH.md.
- Unknown keys filtered out via set intersection before passing to constructor.
- Corrupt JSON / OSError both return default AppSettings.
- `_FILENAME` is module-level constant, not instance state.
