  <action>
**In `_shared_patterns.py`**, add these shared regex patterns and helper functions:

1. `SPEED_RE = re.compile(r'\*\*Speed\*\*\s+(.+)', re.IGNORECASE)` — extracts speed text after **Speed** marker
2. `DICE_IN_TRAIT_RE = re.compile(r'(\d+)\s*\((\d+d\d+(?:[+-]\d+)?)\)\s+(\w+)\s+damage', re.IGNORECASE)` — detects "N (XdY) type damage" patterns in trait text
3. `RECHARGE_RE = re.compile(r'\(Recharge\s+(\d+)(?:\s*[-–]\s*(\d+))?\)', re.IGNORECASE)` — detects "(Recharge X-Y)" in trait names

Add shared helper functions:

4. `detect_dice_in_text(text: str) -> list[DetectedDie]` — scans text with DICE_IN_TRAIT_RE, returns list of DetectedDie with full_match, dice_expr, damage_type, average parsed from groups
5. `detect_recharge(name: str) -> Optional[tuple[int, int]]` — scans trait name with RECHARGE_RE, returns (min, max) tuple or None. For single-value recharge like "(Recharge 6)", return (6, 6).
6. `extract_speed(text: str) -> str` — scans text with SPEED_RE, returns matched speed string or "".

**In each of the three format parsers** (fivetools.py, homebrewery.py, plain.py):

7. Add `_extract_traits(clean_text: str) -> list[Trait]` function:
   - Use `extract_named_section(clean_text, "Traits")` to try extracting explicit Traits section first (this helper already exists from Phase 14-03)
   - If no explicit Traits section found, extract traits from the preamble — the text BEFORE the first section header (use `extract_all_sections(clean_text)` which returns sections dict, get the preamble portion)
   - Split into named blocks using existing `_split_action_blocks()` pattern (or equivalent for each parser)
   - For each block: if it has NO `to_hit_bonus` AND NO `damage_parts` (no match from TO_HIT_RE and HIT_LINE_RE), classify as Trait
   - Create Trait with name, description (text after name), rollable_dice from `detect_dice_in_text()`, recharge_range from `detect_recharge()`
   - Important: skip stat lines (**Armor Class**, **Hit Points**, etc.) — only process entries that start with the bold-italic `***Name.***` pattern (same as action blocks)

8. Add `_extract_speed(clean_text: str) -> str`:
   - Call shared `extract_speed(clean_text)` from _shared_patterns.py

9. Modify existing action extraction to populate `Action.after_text`:
   - In the action parsing logic where `HIT_LINE_RE` is matched, capture everything after the hit/damage match as `after_text`
   - Set `action.after_text = after_text_string` on each parsed Action
   - For actions where HIT_LINE_RE does not match (non-attack actions), after_text remains ""

10. Wire into Monster construction: in each parser's main parse function, call `_extract_traits()` and `_extract_speed()` and pass results to the Monster constructor as `traits=traits, speed=speed`.

**Backward compatibility rules:**
- Old statblocks without explicit "Traits" section: traits extracted from preamble
- Statblocks with no recognizable trait entries: `monster.traits` stays empty list — no error
- Old persisted monsters (already in library): `traits=[]` and `speed=""` via defaults — no breakage
- Actions that already exist in `monster.actions` are NOT moved to traits retroactively for old data
