  - Append the formatted output to the roll output panel using the existing `append_html()` or equivalent method

**4. [[XdY]] auto-average notation rendering (PARSE-08):**

- Create a helper function `_render_double_bracket(text: str) -> str`:
  - Regex: `DOUBLE_BRACKET_RE = re.compile(r'\[\[(\d+)d(\d+)\]\]')`
  - For each match: compute `avg = (n * (s + 1)) // 2` (integer floor of expected value)
  - Replace `[[NdS]]` with `{avg} [[{N}d{S}]]`
  - Per user decision: "[[XdY]] average is always auto-calculated from the dice formula — not editable by the user"
- Apply `_render_double_bracket()` to all displayed text in `show_monster()`:

  - Action raw_text (or the display text used for actions)
  - After-attack-text
  - Any other text that might contain [[XdY]] notation
- This renders in the library detail panel AND the editor preview (both use MonsterDetailPanel)

**5. Add `import re` at top of monster_detail.py** if not already present.

**Important:** The [[XdY]] rendering is display-only — it does NOT modify the underlying Monster data. It's a transformation applied during HTML generation for display.
