**Shared helper in _shared_patterns.py:**

1. Add a `SECTION_BOUNDARY_RE` compiled regex that matches section headers:
   ```python
   SECTION_BOUNDARY_RE = re.compile(
       r'^#{1,3}\s+(?:Actions|Reactions|Legendary Actions|Lair Actions|Bonus Actions)\s*$',
       re.IGNORECASE | re.MULTILINE
   )
   ```
   Also consider the bold-text section header format used in some statblocks: `***Actions***` or `**Actions**` or just `Actions` on its own line. Study the existing parser patterns to see what header formats each parser recognizes, and include those patterns.

2. Add an `extract_named_section(text: str, section_name: str) -> str` function:
   - Find the section start using a pattern like `r'^#{1,3}\s+' + re.escape(section_name)` (and the bold equivalents).
   - Find the next section header after the start using `SECTION_BOUNDARY_RE`.
   - Return only the text between start and next section header (or end of text if no next section).

3. Add `extract_all_sections(text: str) -> dict[str, str]` if helpful — returns a dict mapping section name to section text. This avoids repeated regex scanning.

**Parser updates for plain.py, homebrewery.py, fivetools.py:**

4. In each format parser, find the method that extracts actions (e.g., `_extract_actions()`, `_split_action_blocks()`, or equivalent). Currently these methods operate on the full segment text without respecting section boundaries.

5. Replace the current approach with section-aware extraction:
   - First, extract the "Actions" section text using `extract_named_section(segment_text, "Actions")`.
   - Parse actions ONLY from the Actions section text.
   - Similarly extract "Legendary Actions" and "Lair Actions" sections.
   - Parse legendary_actions from the Legendary Actions section using the same `_split_action_blocks()` / `_parse_action()` logic.
   - Parse lair_actions from the Lair Actions section similarly.

6. In the monster construction step (where the parser builds the `Monster` object), populate `monster.legendary_actions` and `monster.lair_actions` with the parsed results from their respective sections. Regular `monster.actions` comes only from the "Actions" section.

7. If a section header is not found (e.g., no "Legendary Actions" section), the corresponding list stays empty — this is the default.

**Edge cases to handle:**
- Statblocks without explicit section headers (older plain-text formats) — if no "Actions" section header exists, fall back to the current behavior (parse the whole segment as actions) so we don't break existing parsing.
- Section headers that use bold markdown (`**Actions**` or `***Actions***`) instead of `#` headers — check each parser's existing header patterns and include those in the boundary regex.
- "Reactions" section should also act as a boundary (stops Actions from bleeding into Reactions) but Reactions themselves are not parsed into any field in this plan — they just stop the Actions section.
