**BUG-12: Group monsters — damage input doesn't appear**

3. Add a `_on_group_damage_submitted()` handler that implements first-come-first-served damage distribution per user decision:
   - Parse the input as an integer. Negative = damage, positive = healing.
   - For DAMAGE (negative value): iterate group members in display order. For the first non-defeated member (`current_hp > 0`), apply damage. If damage exceeds that member's remaining HP, apply what's left to that member (flooring at 0) and pass the overflow to the next non-defeated member. Continue until all damage is distributed or all members are defeated.
   - For HEALING (positive value): apply healing to the first defeated member (HP=0) or the most damaged member — use a simple approach like healing the first member in order.
   - Emit `damage_entered(member_id, allocated_damage)` for each affected member so the service can apply it.

4. Edge cases: If ALL members are defeated, damage input should be a no-op (or show a brief message). If only one member remains, all damage goes to that member.
