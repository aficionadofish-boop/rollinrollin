**BUG-01: Imported monsters not retained after restart**

In `src/ui/library_tab.py`:
1. Add a `source_files_changed = Signal(list)` signal to `MonsterLibraryTab`.
2. Add an `_imported_paths: set` instance variable initialized to `set()` in `__init__`.
3. In `_process_import_file()`, after successfully importing monsters from a file (the `if result.monsters:` block), add the file path string to `self._imported_paths` and emit `self.source_files_changed.emit(list(self._imported_paths))`.

In `src/ui/app.py`:
4. In the main window class, connect `self._library_tab.source_files_changed` to a handler that saves the list to the persistence service under the `loaded_monsters` category. The handler should call `self._persistence.save('loaded_monsters', paths)` (or the equivalent method name used by PersistenceService).
5. In `_load_persisted_data()`, after loading `_persisted_monsters` from persistence, iterate over each path string and call the parser to re-import those files into the library. Use the same parsing flow as the import button — call `self._library_tab._library.add()` or equivalent for each successfully parsed file. Also populate `self._library_tab._imported_paths` with the loaded paths so subsequent imports don't lose previously saved paths.
6. Handle missing files gracefully: if a persisted path no longer exists on disk, skip it silently (don't crash, don't remove it from the persisted list — the file might be on a removable drive).

**BUG-02: Library search single-result click selects cell, not row**

In `src/ui/library_tab.py`:
7. In `_on_selection_changed()`, replace the use of `selected.indexes()` (which returns per-cell indexes from the selection delta) with `self._table.selectionModel().selectedRows()` (which returns full row indexes regardless of which cell was clicked).
8. If `selectedRows()` is empty, call `self._detail_panel.clear()` and return early.
9. Otherwise, use `rows[0]` as the proxy index, map to source, and load the monster — same as existing logic but with the correct index source.
