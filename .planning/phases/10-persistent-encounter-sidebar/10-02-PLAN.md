**MainWindow integration — this is the critical wiring task.** Modify `src/ui/app.py` to:

   - Add: `from src.ui.load_encounter_dialog import LoadEncounterDialog`

   - Add: `from PySide6.QtCore import Qt` (for `Qt.DockWidgetArea.RightDockWidgetArea`)
   - Add: `import datetime` (for save timestamp)

2. **Create sidebar dock (in `__init__`, after tabs are created):**
   ```python
   self._sidebar = EncounterSidebarDock(library=self._library, parent=self)
   self.addDockWidget(Qt.DockWidgetArea.RightDockWidgetArea, self._sidebar)
   ```

3. **Tab changes:**
   - Replace `self._encounters_tab = EncountersTab(...)` with `self._saves_tab = SavesTab(library=self._library, roller=self._roller)`.
   - Change tab label from `"Encounters && Saves"` to `"Saves"`.
   - Update all internal references from `self._encounters_tab` to `self._saves_tab`.
   - Update `_apply_settings()` to call `self._saves_tab.apply_defaults(settings)`.

4. **Signal rewiring — the old chain was:**
   - `library_tab.monster_added_to_encounter` → `encounters_tab.add_monster_to_encounter`
   - `encounters_tab.encounter_members_changed` → `attack_roller_tab.set_creatures`

   **New chain:**
   - `library_tab.monster_added_to_encounter` → `self._sidebar.add_monster` (Library drop zone → sidebar)
   - `self._sidebar.encounter_changed` → `self._attack_roller_tab.set_creatures` (sidebar → Attack Roller)
   - `self._sidebar.monster_selected` → `self._attack_roller_tab.set_active_creature` (sidebar single-click → preload)
     - Note: `set_active_creature` may not exist yet on AttackRollerTab. If it doesn't exist, add a simple method: find the monster in the creatures list and select it in the creature combo. If AttackRollerTab already has a mechanism to select a creature by name or object, use that.
   - `self._sidebar.switch_to_attack_roller` → `lambda: self._tab_widget.setCurrentWidget(self._attack_roller_tab)` (double-click → switch tab)
   - `self._sidebar.save_btn_clicked` → `self._on_sidebar_save` (sidebar Save button)
   - `self._sidebar.load_btn_clicked` → `self._on_sidebar_load` (sidebar Load button)

5. **Persistence lifecycle for active encounter:**

   **On startup (`_load_persisted_data`):**
   - Change `self._persisted_encounters = self._persistence.load_encounters()` to:
     ```python
     active_enc = self._persistence.load_active_encounter()
     ```
   - After sidebar is created, if `active_enc` is not None and has members:
     - Resolve member names from library: for each `{"name": n, "count": c}` in `active_enc["members"]`, try `self._library.get_by_name(n)` — skip names not found.
     - Call `self._sidebar.set_encounter(active_enc.get("name", ""), resolved_members)`.
     - Track unresolved count and show status bar message if any names couldn't be resolved.

   **On close (`closeEvent`) and autosave (`_autosave`):**
   - Before saving, serialize the sidebar's current state:
     ```python
     members = self._sidebar.get_members()
     if members:
         active_data = {
             "name": self._sidebar.get_encounter_name(),
             "members": [{"name": m.name, "count": c} for m, c in members]
         }
         self._persistence.save_active_encounter(active_data)
     ```
   - Update `_save_persisted_data()` to serialize sidebar state instead of the old `_persisted_encounters`.

   **On sidebar width persistence:**
   - In `closeEvent`, before `event.accept()`, save sidebar width:
     ```python
     self._current_settings.sidebar_width = self._sidebar.width()
     self._settings_service.save(self._current_settings)
     ```
   - On startup, after creating sidebar, restore width:
     ```python
     self._sidebar.set_expanded_width(self._current_settings.sidebar_width)
     ```

6. **Save/Load encounter handlers:**

   `_on_sidebar_save(self)`:
   - Get current encounter from sidebar: `name = self._sidebar.get_encounter_name()`, `members = self._sidebar.get_members()`.
   - If no name, use "Untitled Encounter".
   - If no members, return early (nothing to save).
   - Serialize: `{"name": name, "members": [{"name": m.name, "count": c} for m, c in members], "saved_at": datetime.datetime.now().isoformat(timespec="seconds")}`.
   - Call `self._persistence.save_saved_encounter(saved_data)`.
   - Show status bar message: "Encounter saved".

   `_on_sidebar_load(self)`:
   - Auto-save current encounter first (call `_on_sidebar_save` if sidebar has members).
   - Load saved encounters: `saved = self._persistence.load_saved_encounters()`.
   - If empty, show status bar message "No saved encounters" and return.
   - Show `LoadEncounterDialog(saved, parent=self)`.
   - If dialog accepted:
     - Process deleted indices first (in reverse order to avoid index shifting): call `self._persistence.delete_saved_encounter(idx)` for each.
     - Get selected encounter data from `saved[dialog.selected_index()]`.
     - Resolve names from library.
     - Call `self._sidebar.set_encounter(enc["name"], resolved_members)`.
   - If dialog rejected but deletions were made, still process deletions.

7. **Flush handler update:**
   - Update `_on_flush_category("encounters")` to also clear the sidebar: `self._sidebar.set_encounter("", [])`.

8. **Empty state handling:**
   - Per CONTEXT.md: "When no encounter exists, sidebar toggle is grayed out and not openable."
   - On startup, if no active encounter loaded, the sidebar should start in its disabled/empty state (EncounterSidebarDock handles this internally based on member count).
   - "Sidebar activates when the first monster is dragged onto the drop field" — the `add_monster` call from the Library drop zone signal handles this.
