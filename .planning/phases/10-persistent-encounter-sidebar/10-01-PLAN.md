**PersistenceService (`src/persistence/service.py`):**

The encounters category currently stores `[]` (empty list, never populated by UI). Change the schema to store a dict with `active` and `saved` keys:

```json
{
  "active": {"name": "...", "members": [{"name": "Goblin", "count": 3}]},
  "saved": [{"name": "Ambush", "members": [...], "saved_at": "2026-02-26T14:30:00"}]
}
```

1. Change `_DICT_CATEGORIES` to include `"encounters"` (so empty default is `{}` instead of `[]`).

2. Replace the existing `load_encounters() -> list` and `save_encounters(data: list)` with these methods:
   - `load_active_encounter() -> dict | None`: Loads `encounters.json`, returns the `"active"` key value, or `None` if missing/empty.
   - `save_active_encounter(data: dict) -> None`: Loads current encounters dict, sets `data["active"] = data`, saves back. Uses load+merge pattern (same as `_save_override` in Phase 09-05) to avoid clobbering saved encounters.
   - `load_saved_encounters() -> list`: Loads encounters dict, returns `"saved"` key value or `[]`.
   - `save_saved_encounter(encounter: dict) -> None`: Loads current encounters dict, appends `encounter` to `"saved"` list, saves back.
   - `delete_saved_encounter(index: int) -> None`: Loads current encounters dict, removes `"saved"[index]` by index, saves back.
   - Keep `save_encounters(data)` as an internal method for the flush/autosave lifecycle — it writes the entire dict at once.

3. Update `count("encounters")` to return: `len(saved) + (1 if active else 0)` — so the flush button count reflects both active and saved encounters.

Create `src/ui/encounter_sidebar.py` with `EncounterSidebarDock(QDockWidget)` subclass. This is a self-contained widget — Plan 02 will wire it into MainWindow.

**Class structure:**

```python
from PySide6.QtWidgets import QDockWidget, QWidget, QVBoxLayout, QHBoxLayout, QLabel, QSpinBox, QPushButton, QScrollArea, QSizePolicy, QMenu, QListWidget, QListWidgetItem, QAbstractItemView
from PySide6.QtCore import Qt, Signal, QPropertyAnimation, QEasingCurve
from PySide6.QtGui import QCursor

class EncounterSidebarDock(QDockWidget):
    # Signals (for MainWindow wiring in Plan 02)
    monster_selected = Signal(object)          # Monster — single-click preload
    switch_to_attack_roller = Signal()         # double-click → switch tab
    encounter_changed = Signal(list)           # [(Monster, count)] on any mutation
    view_stat_block_requested = Signal(object) # Monster — context menu "View Stat Block"

    _COLLAPSED_WIDTH = 20
    _DEFAULT_EXPANDED_WIDTH = 300
```

**Key implementation details:**

1. **Constructor** — `__init__(self, library, parent=None)`:
   - Store `self._library` reference for name→Monster resolution.
   - Set `objectName("encounter_sidebar")` — required for potential `saveState`/`restoreState`.
   - Set `setFeatures(QDockWidget.DockWidgetFeature.DockWidgetMovable)` — prevent user from closing or floating.
   - Hide the default title bar: `self.setTitleBarWidget(QWidget())` — we use a custom header instead.
   - Build two internal widgets: `_content_widget` (full panel) and `_handle_widget` (thin strip).
   - Set up `QPropertyAnimation(self, b"maximumWidth")` with 200ms duration and `InOutQuad` easing.
   - Default state: collapsed=False, width=_DEFAULT_EXPANDED_WIDTH.
   - Call `_build_content()` and `_build_handle()`.

2. **`_build_content()`** — the full expanded panel:
   - **Header area** at top with:
     - Encounter name label (read-only, `QLabel`, bold) — text set via `set_encounter_name(name)` method.
     - Summary line: `QLabel` showing "N creatures | XXXX XP" — updated via `_update_summary()`.
     - Button row: Save button (floppy icon or text "Save") and Load button (text "Load") — these emit signals; actual save/load logic lives in MainWindow (Plan 02).
     - Collapse button (e.g. "<<" or a grip icon) aligned right — calls `toggle_collapse()`.
   - **Monster list** — use `QListWidget` with `DragDropMode.InternalMove` for drag-to-reorder:
     - Each item is a `QListWidgetItem` with a custom widget set via `setItemWidget()`.
     - Custom row widget: `QHBoxLayout` with name `QLabel` (expanding), count `QSpinBox` (range 1-99, fixed width 56px), remove `QPushButton` ("X", fixed width 28px).
     - Name format: plain "Goblin" for standard; "Volt the Booyagh (Goblin)" if renamed (per CONTEXT.md). For now, just use `monster.name` — renamed creature support is a Phase 11 concern.
     - Count display: "x3" suffix on name label OR separate count spinbox. Per CONTEXT.md: "Count is inline editable — click the count number to change it directly" → use QSpinBox always visible.
     - Connect `count_spin.valueChanged` to `_on_count_changed(monster_name)` — emit `encounter_changed`.
     - Connect remove button to `_remove_monster(monster_name)`.
     - Connect `QListWidget.itemClicked` → `_on_single_click(item)` → emit `monster_selected`.
     - Connect `QListWidget.itemDoubleClicked` → `_on_double_click(item)` → emit `monster_selected` + `switch_to_attack_roller`.
   - **Context menu** — override `contextMenuEvent` or connect to `customContextMenuRequested`:
     - Items: "Remove", "Remove all {name}", separator, "Roll Attacks", "View Stat Block".
     - "Remove" decrements count by 1 (or removes row if count was 1).
     - "Remove all {name}" removes the entire row.
     - "Roll Attacks" emits `monster_selected` + `switch_to_attack_roller`.
     - "View Stat Block" emits `view_stat_block_requested`.
   - **Empty state**: When no monsters, show a centered `QLabel("No encounter active")` in place of the list. Per CONTEXT.md: "When no encounter exists, sidebar toggle is grayed out and not openable." Implement this by checking member count in `_update_empty_state()`.
   - **Background color**: Apply a subtle background difference via `self.setStyleSheet("EncounterSidebarDock { background-color: #f8f8f8; }")` or similar light gray.

3. **`_build_handle()`** — the collapsed thin strip:
   - A simple `QWidget` with a `QPushButton` showing ">>" or a grip icon, filling the 20px width.
   - Click → `toggle_collapse()`.
   - Initially `setVisible(False)`.

4. **`toggle_collapse()`** — collapse/expand animation:
   - Follow Pattern 2 from RESEARCH.md exactly:
     - If expanding: show content widget, hide handle widget, animate `maximumWidth` from 20 to `_expanded_width`, then in `_after_expand` reset `setMinimumWidth(200)` and `setMaximumWidth(16777215)`.
     - If collapsing: `setMinimumWidth(0)`, animate `maximumWidth` from current width to 20, then in `_after_collapse` hide content widget, show handle widget.
   - Use `_anim.finished.connect` with disconnect pattern (disconnect before reconnecting to avoid stacking).
   - Toggle `self._collapsed` boolean.

5. **Public API** (called by MainWindow in Plan 02):
   - `add_monster(monster: Monster, count: int = 1)` — adds to list or increments existing. Use `blockSignals(True)` on spinbox when setting value programmatically. Emits `encounter_changed`. If this is the first monster (list was empty), auto-expand sidebar and enable toggle.
   - `remove_monster(name: str, all_of_type: bool = False)` — removes one or all. If list becomes empty, auto-collapse and disable toggle.
   - `set_encounter(name: str, members: list[tuple])` — bulk load (for persistence restore). Clear existing, add all. Members are `(Monster, count)` tuples.
   - `get_members() -> list[tuple]` — return `[(Monster, count)]` from current list rows.
   - `get_encounter_name() -> str` — return current encounter name.
   - `set_encounter_name(name: str)` — set the header name label.
   - `set_expanded_width(width: int)` — for restoring persisted width.
   - `save_btn_clicked = Signal()` — emitted when Save button pressed.
   - `load_btn_clicked = Signal()` — emitted when Load button pressed.

6. **XP calculation** — `_update_summary()`:
   - Use the XP table from RESEARCH.md (`_XP_BY_CR` dict).
   - `cr_to_float()` helper for CR sorting.
   - Default sort: CR descending. Sort rows after add/remove by CR.
   - Header shows: "N creatures | XXXX XP". No difficulty rating (requires party data from Phase 11).

7. **Drag-to-reorder**: `QListWidget.setDragDropMode(QAbstractItemView.DragDropMode.InternalMove)` enables item reorder. After drop, re-read the order from `QListWidget` items and emit `encounter_changed`.

**Important anti-patterns to AVOID:**
- Do NOT use `QDockWidget.hide()` for collapse — use width constraints.
- Do NOT store Monster objects for persistence — only use `(monster, count)` tuples at runtime.
- Do NOT import or reference specific tab classes — sidebar is tab-agnostic, signals flow through MainWindow.
- Use `blockSignals(True/False)` on QSpinBox when setting values programmatically to avoid infinite signal loops (Pitfall 5 from RESEARCH.md).
